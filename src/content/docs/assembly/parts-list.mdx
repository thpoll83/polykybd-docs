<Aside>
If you use CNC aluminium cases instead of resin-printed ones, add threaded inserts to the screw holes. Resin prints accept M3 screws directly.
</Aside>

<Aside type="tip">
If a resin case is slightly warped, use a hair dryer to soften it, then press it flat under a heavy object — it really works.
</Aside>

## Optional parts

<Aside type="caution">
Each half has a **single physical expansion slot**. The rotary-encoder header, the pointing-device
connector (trackball/trackpad) and the light-sensor header are **separate, electrically independent
interfaces** that happen to share that one slot — so only **one** component physically fits per
half. Fitting the light sensor therefore replaces the encoder/pointing device on that half. You can
still put a different one on the other half (e.g. the sensor on the left and a trackpad on the
right). Decide which one you want per side before you start.
</Aside>

### Rotary encoder

Either of these fits the PCB footprint:

- **EVQWGD001** — Note: remove the last pin before soldering. This encoder is no longer in production.
- **Alps EC11 encoder**

![EVQWGD001 encoder](../../../assets/assembly/evqwgd001.png)

### Pointing devices (choose one)

| Option | Part | Notes |
|---|---|---|
| Trackball | Pimoroni Trackball | Solder header without the plastic distance holder |
| Trackpad (small) | Cirque TM023023-2024-002 (23mm) | Remove R1 on back to switch to I2C; needs 12-pin FPC ~50–70mm |
| Trackpad (large, experimental) | Cirque TM035035-2024-003 (35mm) | Modify OpenSCAD source before printing the holder |

The 23mm trackpad needs a 3D-printed holder — pick the height that fits your build:

| Holder | STL |
|---|---|
| Slim / low insert | `parts/cirque23_slim_insert_r8.stl` |
| High insert | `parts/cirque23_insert_high_r1.stl` |

Both are in the [PolyKybd hardware repo](https://github.com/thpoll83/PolyKybd) under `parts/`. The OpenSCAD sources (`parts/cirque23_slim_insert.scad`, `parts/cirque23_insert.scad`, and the experimental `parts/cirque35_insert.scad`) are there too if you want to adjust the fit.

Only one pointing device can be installed per half.

### Ambient-light + proximity sensor (highly recommended)

| Option | Part | Notes |
|---|---|---|
| Light + proximity sensor | Pimoroni LTR-559 breakout | Plus a 5-pin 2.54mm pin header. Solders to the sensor's own labelled `3-6V / SDA / SCL / INT / GND` footprint on the Split72 PCB — a separate interface from the encoder and pointing-device connectors in the same expansion slot. |

With the sensor fitted, the keyboard drives keycap brightness from the measured light level and
wakes the displays when your hand approaches — see [Display Brightness](/using/brightness/). It is
optional in firmware (a clean no-op when absent) and side-agnostic. Split72 only — the Split42 has
no expansion port.

<Aside type="caution">
The sensor must be soldered **before** the plate goes on — it does not fit through the plate hole,
so it cannot be added later without an almost complete disassembly. Fit it as the very first
assembly step (see the [Step-by-Step Build Guide](/assembly/step-by-step/)).
</Aside>
