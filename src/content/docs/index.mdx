## What makes it unique

<CardGrid>
  <Card title="72 per-key displays" icon="laptop">
    Every key has its own 72×40 px OLED showing its current function — layers, languages and app overlays are visible at a glance, no guessing.
  </Card>
  <Card title="Fully split ergonomic" icon="puzzle">
    Two halves linked over a full-duplex serial cable, so each hand sits exactly where it feels natural. Available as **Split72** and the smaller **Split42**.
  </Card>
  <Card title="Context-aware overlays" icon="setting">
    The host software tracks your active window and re-labels every key automatically when you switch apps.
  </Card>
  <Card title="Open hardware & firmware" icon="open-book">
    KiCad schematics, gerbers, STL case files, a QMK firmware fork and the Python host app — every layer is open.
  </Card>
</CardGrid>

## The ecosystem

PolyKybd is built from a handful of open-source repositories that work together:

| Component | What it is |
|---|---|
| [PolyKybd hardware](https://github.com/thpoll83/PolyKybd) | KiCad PCBs, 3D-printable case and key stems, assembly guide |
| [QMK firmware](https://github.com/thpoll83/qmk_firmware) | The keyboard firmware (RP2040) — branch `PolyKybd`, variants `split72` & `split42` |
| [PolyKybdHost](https://github.com/thpoll83/PolyKybdHost) | Python host app (Windows / macOS / Linux) — daemon + tray GUI + `polyctl` CLI |
| [Adafruit GFX (fork)](https://github.com/thpoll83/adafruit-gfx-library) | Rendering library + `fontconvert` used to draw icons and text on the displays |
| [PolyKybd CTND](https://github.com/thpoll83/polykybd-ctnd) | Raspberry Pi hardware-in-the-loop test & CI rig (development infrastructure) |

See [Architecture & Ecosystem](/introduction/architecture/) for how these pieces talk to each other.
